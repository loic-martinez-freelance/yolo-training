# Dataset folder structure

Each new dataset project must respect the following structure to be compatible with the training scripts.

Let's take the example of a project named `my_project`. The folder must be placed in the `datasets/` directory.

## File hierarchy

```text
datasets/
└── my_project/                 <-- Your project folder name
    ├── roboflow/               <-- Roboflow export (do not rename this folder)
    │   ├── data.yaml           <-- YOLO configuration file (paths, class names)
    │   ├── train/              <-- Training images and labels
    │   ├── valid/              <-- Validation images and labels
    │   └── test/               <-- Test images and labels (optional)
    │
    └── train_params.yaml       <-- (Optional) Training configuration file
```

## Important file details

### 1. `roboflow/data.yaml`

This file is automatically generated when exporting from Roboflow (YOLOv11 or YOLOv8 format). It is **essential** as it tells the model where to find images and which classes to detect.

### 2. `train_params.yaml` (Optional)

This file allows you to customize training hyperparameters specifically for this project, without modifying the global code. If this file is missing, the default values from the `train.py` script will be used.

**Example content:**

```yaml
epochs: 200 # Number of epochs (default: 100)
patience: 20 # Early stopping if no improvement (default: 10)
batch: 16 # Batch size (default: 8)
imgsz: 640 # Image size (default: 640)
```
